# 中英逐句对照真题文档制作

## 这个技能解决什么

用户通常有两份 PDF：一份**英文试题**（完形、阅读、翻译、写作题），一份**参考答案**
（选择题答案 + 翻译参考译文 + 写作范文）。目标是产出一份排版舒适、**英文一句紧跟中文一句**、
从头到尾完整覆盖的 `.docx` 学习材料，并把两份文件"结合"起来。

关键认知（一定先想清楚再动手）：**答案 PDF 往往只给选择题的字母/单词和翻译题的参考译文，
不会给完形、阅读、新题型本文的中文翻译。** 所以"结合两个文件"只能直接对齐翻译那一节；
其余本文的中文需要由你（Claude）逐句翻译。动手前要把这一点跟用户讲清楚，避免误以为答案 PDF
里有全部译文。

## 整体流程

1. **抽取两份 PDF 的文本**（用 `scripts/extract_pdf_text.py`，输出 UTF-8 txt 再用 Read 通读）。
   不要直接在终端 cat 中文，Windows 下会乱码。
2. **读懂结构**：定位完形本文与 1–20 选项、阅读 Text1–4 本文与题目、Part B 本文与匹配项、
   翻译题英文与参考译文、写作要求与范文。
3. **逐句对齐 + 翻译**：把英文按句切分，每句配一句中文。
   - 翻译题：直接用答案 PDF 的**官方参考译文**。
   - 其余本文：你来翻译（忠实、通顺、句对句）。
   - 写作范文：原样收录英文，并逐句给中文。
4. **写成内容规格 JSON**（schema 见下），把所有块按顺序排好。
5. **渲染**：`python scripts/render_bilingual_docx.py spec.json 输出.docx`。
   排版（字体/行距/留白/选项并排/正解红✔）全部由脚本固定，你只管内容。
6. **自查完整性**：核对完形 1–20、阅读题号无缺；说明哪些是官方译文、哪些是你翻译的；
   OCR 讹误处加注。

## 已经定型的排版约定（用户反复确认过，保持一致）

- **大标题**精简，例如 `2026 年（英语二）`，不要带"全国硕士研究生招生考试"这类长定语。
- **逐句对照**：英文一段、紧跟中文一段；**句对之间留白**，整体行距 1.25、正文 11pt、页边距适中——
  目标是"看起来舒服、不拥挤"，不要为省页把它压得密密麻麻（除非用户明确要省页）。
- **完形填空**：把 1–20 空**用答案单词填进本文**，让英文成为完整可读、可翻译的句子；
  20 道选项**4 个并排成一行**（英一行、中一行），因为完形选项是单词，并排清楚又省地方。
- **阅读题选项**：每行放 **2 个**（英一行、中一行）。阅读选项是整句，4 个并排会折行错乱，2 个正好。
- **正确答案**用**红色加粗 + ✔** 标出（颜色由脚本统一处理）。
- **Part B 格式逐年不同，先看清再选块**：
  - **小标题/人物匹配**（如 2026）：用 `sub` 块，把**题号标注到对应待选项后面**（命中项红色
    `←【第42题 姓名】✔`，未选中项标"（多余干扰项）"），不要单开"答案匹配"一节。
  - **判断正误 T/F**（如 2010）：用 `numbered` 块，每条 `ans` 填 "T"/"F"。
  - **排序/段落对应**等其他变体：同样用 `numbered` 块，`ans` 填该题答案字母。
  先读真题里 Part B 的 Directions 判断属于哪种，再决定用哪个块。
- 顶部放一行斜体小字**说明**：哪些是官方填空/译文、哪些是整理者翻译、✔ 表示正确答案。

## 内容规格 JSON

完整的块类型、字段和示例见 `scripts/render_bilingual_docx.py` 文件顶部的 docstring，以及
`references/example_spec.json`（一个可直接渲染的最小示例）。核心块：

- `pair`：一句英文 + 一句中文（完形本文、阅读本文、Part B 本文、翻译、范文都用它）
- `cloze_opts`：完形某题的 4 个选项（4 并排）
- `qhead` + `read_opts`：阅读题干 + 选项（2 并排）
- `sub`：Part B 待选小标题，带题号标注
- `h1/h2/h3`、`note`：分节标题与小字说明

切句提示：以英文句号/问号/引号结束为界，一句一个 `pair`。引号内的完整句可与引导语合并为一句，
以免碎成太多小段。

## 范围沟通（动手前确认）

不同用户要的范围不同，先确认再做，避免返工：
- 是否包含**题目和 ABCD 选项**（多数要包含），还是只要本文。
- 是否要把**写作 Directions / 图表**也放进去（图表是 PDF 里的图片，可用 PyMuPDF 抽出插入；
  有用户会说"没用的不补"——那就略过）。
- 输出格式默认 `.docx`；若用户要 Excel 左右两列或 Markdown，可另行调整（本技能聚焦 Word）。

## 依赖

需要 Python 包 `python-docx` 和 `PyMuPDF`(fitz)。若缺，先 `pip install python-docx pymupdf`。
文件被 Word 打开时无法覆盖，会报 PermissionError——此时改输出文件名或请用户先关闭。
